User Manual

The application has the following functionalities:
  • Generate: generates a new set of points; the valid number of points is in range [3; 256].
  • Speed: sets the speed fo the simulations.
  • Stop: stops the simulation.
  • Clear log: clears the simulation log.
  • Naïve: starts the naïve algorithm simulation, with the following color coding:
    • Green: part of the convex hull
    • Magenta: edge being tested
    • Red: point on the right
    • Blue: point on the left
  • Smart Naïve: starts the naïve algorithm simulation. The color coding is the same as Naïve.
  • Gift Wrapping: starts the gift wrapping algorithm simulation, with the following color coding:
    • Green: part of the convex hull
    • Cyan: processed points
    • Yellow: leftmost turn
  • QuickHull: starts the QuickHull algorithm simulation, with the following color coding:
    • Green: part of the convex hull
    • Magenta: convex hull lower bound
    • Red: point on the right
    • Blue: point on the left
    • Cyan: processed points
    • Yellow: furthest point
  • Monotone Chain: starts the monotone chain algorithm simulation, with the following color coding:
    • Green: part of the convex hull
    • Magenta: current convex hull attempt
  • Graham Scan: starts the graham scan algorithm simulation, with the following color coding:
    • Blue: pivot
    • Green: part of the convex hull
    • Magenta: current convex hull attempt
  • Kirkpatrick-Seidel: starts the Kirkpatrick-Seidel algorithm simulation, with the following color coding:
    • Green: segment of the convex hull
    • Black Line: current line the bridge is being searched for
    • Magenta: current points considered in the BRIDGE function
    • Blue: points to the left of the bridge
    • Red: points to the right of the bridge
    • Yellow: discarded points
